Git

Git是一个开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。 Git是用C语言写成，它是目前世界上最先进的分布式版本控制系统。

安装Git

下载地址：https://git-scm.com/downloads

配置Git

配置命令：

git config --global user.name "your name"
git config --global user.email "your email"
git config --global core.editor "your editor"
git config --global merge.tool "your merge tool"
git config --global diff.tool "your diff tool"

设置Git的用户名和邮箱,默认编辑器和合并工具(这两个配置是可选的)。
--global参数是全局配置，也就是对当前用户起作用，而不用在每个仓库单独配置。

创建版本库

创建一个版本库非常简单，只需要在命令行中输入以下命令：

mkdir myproject
cd myproject
git init

这将创建一个名为myproject的文件夹，进入该文件夹，执行git init命令，这将创建一个名为.git的子文件夹，这个文件夹是版本库。
如果已经提前有了文件夹，也可以直接进入该文件夹，执行git init命令。

添加文件

git add filename
git add . // 添加所有文件

这将把文件添加到暂存区。

提交文件到本地仓库

git commit -m "commit message"

这将把暂存区的文件提交到本地仓库，并附带提交信息。

查看状态

git status

这将显示当前仓库的状态，包括暂存区、工作区、版本库。

查看日志

git log

版本回退

git reset --hard commit_id

这将回退到指定提交的版本。

远程仓库

远程仓库是指托管在远程服务器上的版本库，可以让多个开发者共同开发一个项目。

git remote add origin <远程仓库URL>.git

这将把本地仓库和远程仓库关联起来。

推送本地仓库到远程仓库：

git push origin main

这将把本地仓库的main分支推送到远程仓库的main分支。

从远程仓库拉取代码：

git pull origin main

这将从远程仓库的main分支拉取代码到本地仓库的main分支。

分支管理

分支是用来开发新功能或修复bug的，在版本库中创建分支：

git branch new_branch

这将创建一个名为new_branch的分支。

切换分支：

git checkout new_branch

这将切换到名为new_branch的分支。

合并分支：

git merge new_branch

这将把new_branch分支合并到当前分支。

删除分支：

git branch -d new_branch

这将删除名为new_branch的分支。

删除工作树

如果不再需要某个工作树，可以执行以下命令：

git worktree remove D:/code/html

这将删除名为D:/code/html的工作树。

Git无法获取CA证书

1. 更新或安装 SSL 证书
2. 配置 Git 使用不安全的连接（不推荐）
如果你处于测试环境或完全信任该网络，你可以临时配置 Git 忽略 SSL 证书验证。这可以通过设置 Git 配置来实现：

git config --global http.sslVerify false

注意：这种方法会降低安全性，因为它允许 Git 在没有验证 SSL 证书的情况下进行通信。

3. 指定一个自定义的 CA 证书路径
如果你信任某个 CA 证书，你可以通过设置 Git 配置来指定一个自定义的 CA 证书路径：

git config --global http.sslCAInfo /path/to/custom_ca_file.pem

使用 SSH 而不是 HTTPS

数组方法

push: 向数组末尾添加元素

shift: 删除数组开头的元素

unshift: 向数组开头添加元素

var arr = [1, 3, 2];
arr.push(4); // [1, 3, 2, 4]
arr.pop(); // 4
arr.shift(); // 1
arr.unshift(0); // [0, 3, 2]
arr.splice(1, 1, "apple"); // [0, "apple", 2]
arr.sort(); // [0, 2, "apple"]
arr.reverse(); // ["apple", 2, 0]
arr.join("-"); // "apple-2-0"

// 箭头函数
arr.forEach((e) => { console.log(e); });

var str = "Hello, world!";
console.log(str.charAt(0)); // "H"
console.log(str.charCodeAt(0)); // 72
console.log(str.concat(" How are you?")); // "Hello, world! How are you?"
console.log(str.indexOf("l")); // 2
console.log(str.lastIndexOf("l")); // 9
console.log(str.localeCompare("hello")); // 1
console.log(str.match(/l/g)); // ["l", "l"]
console.log(str.replace("l", "L")); // "HeLLo, world!"
console.log(str.search("l")); // 2
console.log(str.slice(2, 5)); // "llo"
console.log(str.split(" ")); // ["Hello,", "world!"]
console.log(str.substring(2, 5)); // "llo"
console.log(str.toLowerCase()); // "hello, world!"
console.log(str.toUpperCase()); // "HELLO, WORLD!"
console.log(str.trim()); // "Hello, world!"

JSON

JSON(JavaScript Object Notation)是一种轻量级的数据交换格式，它基于ECMAScript的一个子集。它是一种基于文本的、可读性高的、占用空间小的的数据交换格式。

JSON的创建


var obj = {name: "John", age: 30, city: "New York"};
var json = JSON.stringify(obj);
console.log(json); // {"name":"John","age":30,"city":"New York"}

JSON的解析


var json = '{"name":"John","age":30,"city":"New York"}';
var obj = JSON.parse(json);
console.log(obj.name); // "John"

JSON.stringify()方法用于将JavaScript对象转换为JSON字符串。

JSON.parse()方法用于将JSON字符串转换为JavaScript对象。


var obj = {name: "John", age: 30, city: "New York"};
var json = JSON.stringify(obj);
console.log(json); // {"name":"John","age":30,"city":"New York"}
var obj2 = JSON.parse(json);
console.log(obj2.name); // "John"

Nginx

Nginx 是一个高性能的 HTTP 和反向代理服务器.

安装

下载安装包：

https://nginx.org/en/download.html

配置

打开 nginx.conf 文件，找到 http 部分，修改配置：

http {
    server {
        listen       80;
        server_name  localhost;

        location / {
            root   html;
            index  index.html index.htm;
        }

        location /api {
            proxy_pass http://localhost:3000;
        }
    }
}

解释：

  • listen：监听端口号
  • server_name：域名
  • location /：配置网站的默认访问路径

启动

打开命令行，切换到 nginx 安装目录，输入命令：

start nginx

测试

打开浏览器，输入地址：

http://localhost/api/

如果看到返回的数据，说明配置成功。

Maven依赖管理

依赖配置

Maven依赖网站：https://mvnrepository.com/


<dependency>
    <groupId>junit</groupId>
    <artifactId>junit</artifactId>
    <version>4.12</version>
    <scope>test</scope>
</dependency>

依赖传递


<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-core</artifactId>
    <version>4.3.11.RELEASE</version>
</dependency>
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-context</artifactId>
    <version>4.3.11.RELEASE</version>
</dependency>
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-web</artifactId>
    <version>4.3.11.RELEASE</version>
</dependency>
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-webmvc</artifactId>
    <version>4.3.11.RELEASE</version>
</dependency>
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-test</artifactId>
    <version>4.3.11.RELEASE</version>
    <scope>test</scope>
</dependency>

解释：groupId：依赖的groupId；artifactId：依赖的artifactId；version：依赖的版本号；scope：依赖的范围，可选值有compile、test、provided、runtime、system。

排除依赖


<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-core</artifactId>
    <version>4.3.11.RELEASE</version>
    <exclusions>
        <exclusion>
            <groupId>commons-logging</groupId>
            <artifactId>commons-logging</artifactId>
        </exclusion>
    </exclusions>
</dependency>

解释：exclusions：排除依赖的groupId和artifactId。

依赖范围


<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-core</artifactId>
    <version>4.3.11.RELEASE</version>
    <scope>compile</scope>
</dependency>

解释：scope：依赖的范围，可选值有compile、test、provided、runtime、system。
scope的作用范围：compile（默认）、test、provided(主程序，测试时运行)、runtime(测试，打包时运行)、system。

Maven的生命周期

clean：清理项目

default：默认的生命周期，相当于clean、site、validate、compile、test、package、verify、install、deploy。

site：生成项目报告

compile：编译项目

test：测试项目

package：打包项目

install：安装项目

deploy：部署项目
